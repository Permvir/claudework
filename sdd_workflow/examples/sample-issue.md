# 示例：添加用户头像上传功能

以下是一个按照 SDD 规范编写的 issue 示例。

---

## 背景

目前用户个人资料页面不支持自定义头像，所有用户显示默认头像。用户反馈希望能上传自己的头像图片。现有的个人资料编辑接口已支持修改昵称和简介，头像上传是在此基础上的扩展。

## 需求

1. 用户资料编辑页面增加头像上传入口，支持点击和拖拽上传
2. 支持 JPG、PNG 格式，文件大小不超过 2MB
3. 上传后自动裁剪为 1:1 正方形，生成 200x200 和 80x80 两个尺寸
4. 上传成功后实时预览，保存后全站生效
5. 提供"恢复默认头像"选项

## 验收标准

- [ ] 用户可以在编辑资料页面上传头像图片
- [ ] 超出格式或大小限制时显示明确的错误提示
- [ ] 上传完成后页面立即显示新头像，无需刷新
- [ ] 导航栏和评论区等位置同步显示新头像
- [ ] 可以重置为默认头像
- [ ] 上传接口有频率限制（每分钟不超过 5 次）

## 关联 Issue

- http://gitlab.example.com/mygroup/backend/-/issues/15 — 后端文件上传服务重构
- http://gitlab.example.com/mygroup/common/-/issues/3 — OSS SDK 升级

## 技术备注

- 图片存储使用现有的 OSS 服务，bucket: `user-avatars`
- 裁剪使用前端 `cropperjs` 库，服务端不做二次处理
- 头像 URL 加入版本号参数避免缓存问题：`avatar.jpg?v={timestamp}`
- API 端点复用现有的 `PUT /api/v1/users/profile`，增加 `avatar` 字段

## 测试计划

- 单元测试：头像 URL 生成、文件格式校验、大小校验
- 集成测试：上传 → 裁剪 → 保存 → 读取完整流程
- 边界测试：超大文件、不支持的格式、并发上传
